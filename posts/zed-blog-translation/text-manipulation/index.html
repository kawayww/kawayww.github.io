> 步法: 定位 & 选区

命令面板(command palette)是您探索新命令并学习对应快捷键的最佳资源之一: 用 ctrl+shift+p 打开它
它允许模糊搜索, 因此您可以键入待查找命令中的几个字符, 以快速找到它

在面板中, 如果您搜索 editor: move 或 editor: select, 您将发现大量命令: 跳转到文件开头/行尾/单词开头等, 选择下一个单词/下一行/下一段等
直接技艺所有所有这些命令既令人生畏也无必要, 我们将专注于学习光标定位和选择的基本功, 这些基础块可以直观地组合起来, 以实现更复杂的操作

> 方向键

  • left: 向左移动光标
  • right: 向右移动光标
  • up: 向上移动光标
  • down: 向下移动光标

> 修饰符

  • cmd: 跳转到编辑器中的某个 “边缘”
    例如: cmd-left 跳转到当前行的开头, cmd-up 跳转到文件的开头, 你能猜出 cmd-right 和 cmd-down 是做什么的吗?
  • alt: 在被以 “单词” 为分割单位的文本间跳转
    当遇到 non-alphanumeric(非字母与数字) 的字符(如空格/标点), 通常会出现单词边界
    例如: alt-left 跳转到当前单词开头, alt-right 跳转到当前单词末尾, alt 与 ctrl 组合时可在 subword(子词) 间跳转, 这对驼峰或 snake_case 命名的变量很有用
  • shift: 将 “选择” 置换为 “选择”
    例如: shift+left 向左选择, 而不是简单移动, 其可与其他修饰符组合使用, cmd-shift-left 以选择行首

> 技法

> 剖析

  1. 定位到第一个 " 的位置
  2. 通过 editor: select right 选择 "
  3. 通过 editor: select all matches 选择所有实例
  4. 通过 editor: delete 删除所有 "
  5. 通过 editor: cancel 退出多光标模式
  6. 通过 editor: select left 选择前面的 ,
  7. 选择 , 的所有实例
    我们在这里不会使用 editor: select all matches, 因为我们不想选择分割列名的逗号
    相反, 我们按住对应 editor: select next 的快捷键, 重复选择下一个匹配项 (editor: undo selection 可以撤销选择)
  8. 通过 editor: move right 退出选择单格的状态状态
  9. 通过 editor: select to next word end 直接选择在逗号后面的所有单词(这里会包括一些整数, 但无所谓)
  10. 通过 editor: convert to title case 转换为首字母大写

> 回顾所选

您可能希望在继续处理前查看所有被选择的文本, 以确保某个选择没抓取不该选中的内容
虽然可以滚动当前文件来复查所选内容, 但如果它们分布稀疏杂乱且位于长文件中, 这会变得相当繁琐乏味
此时运行 editor: open selections in multibuffer 会非常高效, 它能将所有选区集中显示在紧凑的多缓冲区视图中
通过多缓冲区视图中, 您可以更轻松地复查所选, 进行编辑, 还有保存更改
