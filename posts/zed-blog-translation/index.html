『Zed's blog』
摘要
这里是对 Zed’s blog 中部分博客的翻译

Zed 是一个用 rust 编写的现代化的代码编辑器
值得一提的是, 其开发人员还是 Atom, Xi, tree-sitter 的作者
啊对, 没错, Atom 与 Xi 就是那两曾声名大噪但现在死了的编辑器, 咳咳

已发布

› Rope & SumTree

梳理 Zed 的 rope / SumTree 文本结构 为什么不用 string
以及 SumTree 如何维护长度, 行列, UTF-16 摘要
以及它如何支持高效编辑, 快照与坐标换算

› CRDTs 让协作文本编辑成为 Zed 的核心

说明 Zed 的协作编辑如何建立在 CRDT 之上 用 insertion id 与 offset 表示逻辑位置
用 tombstone 与版本向量处理删除, 用 Lamport 时间戳保证并发插入的顺序一致

› 基于 Tree-Sitter 生成语法感知的任务

介绍 Zed 的 Task 系统 一次性命令, tasks.json 模板, 上下文变量
以及如何用 Tree-sitter 把测试函数等语法节点暴露为可运行任务

› 文本处理功夫 の 黑带进阶之路

用几组练习串起 Zed 的文本操作能力 定位, 选区, 多光标与大小写转换
行操作, 以及如何把这些基础动作组合成批量编辑流程

› 文本坐标系

梳理 Zed 中一系列坐标类型的含义与使用场景
探讨缓冲区/LSP/Display, 以及协作编辑中, 坐标的稳定锚点
