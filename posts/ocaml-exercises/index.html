> 从哪里开始

ocaml-exercises 是 OCaml 官网整理的一套习题
它的灵感来自 Ninety-Nine Lisp Problems, 题目按照主题分成了 Lists, Arithmetic, Binary Trees, Graphs 等若干类, 并且每道题都标了 Beginner, Intermediate, Advanced 三档难度
官方页面同时给出了 statement, 简单测试, 以及一份参考解答, 所以它很适合拿来做一套循序渐进的入门练习

这篇先不贪多, 只从 Lists 分类中最开头的几题开始
这些题本身都不算难, 但很适合用来熟悉 OCaml 里最基础的几样东西:

  • pattern matching
  • 递归
  • option 类型
  • tail recursion

如果你还没装本地环境, 可以先按官网指引安装 OCaml, 也可以直接在浏览器里用官方 playground 先跑起来
我这里更关心的是写法本身, 所以下面的代码都尽量保持简单, 不急着秀技巧

> 为什么这组题适合入门

很多语言的算法题一上来就喜欢把人往数组, 哈希表, 双指针上推
而 ocaml-exercises 的开头几题更像是在训练另一件事:
当数据结构已经明确是 list 时, 你能不能自然地用 “拆开再讨论” 的方式去写程序

例如看到 list, 你几乎总可以先问:

  • 它是不是空表
  • 它是不是只剩一个元素
  • 它是不是能拆成 x :: xs

这种思路在 OCaml 里非常自然, 因为 pattern matching 本来就是最顺手的表达方式
也正因为如此, 这批题比起 “刷题感”, 更像是在练一门语言的基本手感

> Lists 里的前 6 题

> 1. 取 list 的最后一个元素

第一题要求写出:

'a list -> 'a option

也就是给一个 list, 返回最后一个元素, 如果 list 为空, 就返回 None

我更喜欢直接按 list 的形状来写:

let rec last = function
  | [] -> None
  | [x] -> Some x
  | _ :: xs -> last xs

这里最关键的是第二个分支 | [x] -> Some x
它表示 “当前 list 只剩最后一个元素了”, 于是递归可以在这里结束

这个写法的好处是很直白
你不需要维护额外状态, 只需要不断把问题缩小成 “求尾部 list 的最后一个元素”

> 2. 取 list 的最后两个元素

第二题和第一题几乎是同一个套路, 只是返回值从一个元素变成两个元素组成的 pair:

let rec last_two = function
  | [] | [_] -> None
  | [x; y] -> Some (x, y)
  | _ :: xs -> last_two xs

和前一题相比, 这里多注意一个 case:

| [] | [_] -> None

因为长度小于 2 的 list 都不满足题意
而一旦命中 | [x; y], 说明我们已经把问题缩到了最小可解规模

这两题放在一起看, 很适合体会一件事:
当题目本身就是在问 “list 的末尾长什么样” 时, 递归其实是在不断丢掉前缀

> 3. 取第 n 个元素

第三题开始引入 “位置” 这个概念
官方这里采用从 0 开始计数的方式, 所以 at 2 ["a"; "b"; "c"; "d"] 的结果应该是 Some "c"

let rec at k = function
  | [] -> None
  | x :: xs ->
      if k = 0 then Some x
      else at (k - 1) xs

这题的核心在于: 每递归一次, 我们同时做两件事

  • list 去掉头部一个元素
  • 下标 k 减 1

直到 k = 0, 说明当前头元素就是我们要找的那个位置

这里顺手也能看出 option 的一个好处:
越界时自然返回 None, 不需要像某些库函数那样抛异常

> 4. 求 list 长度

这一题官方顺手提了一个 bonus: 最好写成 tail-recursive
原因也很直接, tail recursion 可以在 list 很长时保持常量级栈空间

let length xs =
  let rec aux acc = function
    | [] -> acc
    | _ :: rest -> aux (acc + 1) rest
  in
  aux 0 xs

这里的 acc 是 accumulator, 用来保存目前已经数过多少个元素
aux 每处理一个节点, 就把 acc + 1 传给下一轮递归

这一题值得记住的不是 length 本身, 而是这种写法:

let f xs =
  let rec aux acc = function
    | ...
  in
  aux init xs

后面写 rev, map, filter, 甚至很多树和图的遍历时, 这个骨架都非常常见

> 5. 反转 list

反转 list 也是一个非常标准的 accumulator 练习题
如果你已经接受了上一题的思路, 那这题几乎就是顺着写出来的:

let rev xs =
  let rec aux acc = function
    | [] -> acc
    | x :: rest -> aux (x :: acc) rest
  in
  aux [] xs

关键就在这一句:

| x :: rest -> aux (x :: acc) rest

每次把当前头元素接到 acc 前面, 最终 acc 就会变成原 list 的逆序结果

这题也很适合和命令式语言的写法做对比
在命令式语言里, 大家容易先想 “我需要一个下标, 然后从后往前取”
但在 OCaml 里, 用 x :: rest 去消费输入, 再用 x :: acc 去构造输出, 往往是更自然的思路

> 6. 判断回文

前面几题做完之后, 第六题其实就很顺了
题目要求判断一个 list 是否是 palindrome, 也就是正着看和反着看完全一样

最直接的写法就是:

let is_palindrome xs =
  xs = List.rev xs

这当然不是最讲究性能的方案, 因为它会额外做一次 List.rev
但对入门题来说, 它胜在表达非常清楚:
“回文” 这件事, 本来就等价于 “它和它的逆序相等”

这一题很适合提醒自己一件事:
并不是所有题都应该一上来就追求最花的解法
如果某个定义本身就足够直接, 那先把定义翻译成代码, 往往就是最好的起点

> 小结

只看这前 6 题, 其实已经能看出 OCaml 练习的一个节奏:

  • 先把输入拆成几种 shape
  • 再决定哪些 case 是 base case
  • 最后让递归去把问题缩小

如果你之前更习惯命令式思维, 那么这几题最值得练的地方, 往往不是 “写出答案”, 而是把眼睛训练成一看到 list, 就会自然去想 [], [x], x :: xs 这些分支

下一步如果继续写, 比较自然的推进方式就是把 flatten, compress, pack, encode 这几题接上
它们会开始引入自定义代数数据类型, 以及更明显的 “先累积, 后反转” 这种写法, 也会比前面的题更有一点函数式编程练习题的味道
