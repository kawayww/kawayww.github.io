我们很高兴宣布 GNU Guix 1.3.0 的发布!

本次发行包含:

  • ISO-9660 安装镜像
  • 虚拟机镜像
  • 可在现有 GNU/Linux 发行版上安装包管理器的 tar 包, 既可以 从源码安装, 也可以 从二进制安装

Guix 用户可以通过 guix pull 完成更新

距离上个版本已过去将近 6 个月, 期间有 212 位贡献者提交了代码与软件包, 还有许多人参与了 code review, 系统维护, 翻译, 网站更新, Outreachy mentoring 等其他重要工作

在这段时间里, 仓库新增了 8300 多次提交, 下面我们会尽量简明地概括其中最值得注意的变化

> 用户体验

Guix 的一个鲜明特性是对 declarative deployment 的支持
与其不断执行 guix install 和 guix remove, 你也可以使用 guix package --manifest=manifest.scm, 在 manifest.scm 中声明你想安装的软件, 例如:

;; This is 'manifest.scm'.
(specifications->manifest
  (list "emacs" "guile" "gcc-toolchain"))

这样会精确安装文件中列出的那些包, 你可以把它纳入版本控制并分享给其他人
过去用户需要手写 manifest, 这虽然不算困难, 但对刚想转向 declarative model 的人来说始终是个门槛

新的 guix package --export-manifest 命令, 以及配套的 --export-channels, 现在可以基于 现有 profile 的内容自动生成 manifest
这让用户更容易从 “按需运行 guix install” 的 imperative model, 过渡到更加正式的 declarative model, 这项能力大家已经期待很久了

喜欢第一时间尝鲜自由软件最新上游版本的用户, 应该会很喜欢新的 --with-latest package transformation option
它复用了 guix refresh 的那套逻辑, 会查找某个包的最新 upstream release, 下载, 认证并尝试构建它
这在新版本还没被 Guix 打包时尤其有用, 例如下面的命令会尝试安装更高版本的 QEMU:

$ guix install qemu --with-latest=qemu
The following package will be upgraded:
  qemu 5.2.0 -> 6.0.0
...

当然这并不保证一定成功, 如果新版本改了构建系统, 或引入了当前 Guix 配方里没有声明的新依赖, 构建仍可能失败
但它给了用户额外的灵活性, 对开发者来说也能快速检查某个软件包是否能在其依赖的最新版之上顺利构建

为了改善整体体验, 还有不少零散但重要的打磨
例如新增了 --verbosity 级别, 默认 --verbosity=1 会减少下载阶段输出的细节, 更符合大多数用户的预期

另一个实用改进是输入错误时的自动建议:

$ guix package --export-manifests
guix package: error: export-manifests: unrecognized option
hint: Did you mean `export-manifest'?

$ guix remve vim
guix: remve: command not found
hint: Did you mean `remove'?

如果你正在通过 SSH 配置 build offloading, 也会发现流程更简单了
目标机器不再需要把 guile 可执行文件放进 PATH 并额外配置 GUILE_LOAD_PATH, 现在 offloading 会统一通过 guix repl 完成相应操作

Guix 参考手册如今已完整翻译为 法语, 德语 与 西班牙语, 另有 更多语言的初步翻译, 包括 俄语 与 中文
Guix 自身也已完整翻译为法语, 德语与斯洛伐克语, 并在近 20 种语言上提供部分翻译
所有翻译工作现在都通过 Weblate 管理, 也欢迎你 参与

> 开发者工具

对 packager 来说也有不少好消息
首先, guix import 新增了 Go 的递归 importer, 可以为整批 Go 包生成 package definition 或模板
与此同时, 面向 Rust 包的 guix import crate 在递归模式下也开始遵循 “semantic versioning”

guix refresh 现在包含新的 updater: sourceforge 与 generic-html
前者用于托管在 SourceForge 上的软件, 后者则会扫描软件主页来检测更新, 这显著扩大了 guix refresh 的覆盖范围

开发者还会喜欢新的 --with-patch 选项, 它允许你在一批软件包的构建过程中, 对其中一个或多个包应用特定 patch

在 1.2.0 中引入的 Guix System image API 基础上, guix system vm-image 与 guix system disk-image 现在被统一的 guix system image 取代
例如:

guix system vm-image --save-provenance config.scm

现在写成:

guix system image -t qcow2 --save-provenance config.scm

而:

guix system disk-image -t iso9660 gnu/system/install.scm

现在写成:

guix system image -t iso9660 gnu/system/install.scm

这种统一带来了更好的性能, 因为生成镜像时不再需要先启动一个虚拟机, 底层工作由专门的 genimage 工具完成
另一个好处是 qcow2 镜像现在默认已压缩, 不再需要再通过 xz 之类的工具手工做一次后处理
如需了解更多, 可阅读 guix system image 文档

最后, 新增的 GUIX_EXTENSIONS_PATH 搜索路径能让 Guix 扩展, 例如 Guix Workflow Language, 自动暴露自身的 Guile 模块, 从而显著简化部署流程

> 性能

你很可能已经注意到 substitutes 的安装速度变快了, 也就是预构建二进制包的下载速度变快了
正如我们此前在 文章 中解释的那样, 这部分得益于对 zstd 压缩的 opportunistic use, 它在解压吞吐上表现出色
如今 daemon 与 guix publish 也已经支持 zstd, 与 gzip, lzip 并列

另一个有助于加速 substitutes 获取的改动是 local substitute server discovery
guix-daemon 新增的 --discover 选项 会通过 Avahi, 利用 mDNS/DNS-SD 协议自动发现并使用局域网内公布的 substitute server
类似地, guix publish 新增的 --advertise 选项 则可把自己广播到局域网中

在 Guix System 上, 你可以临时执行:

herd discover guix-daemon on

也可以直接在 系统配置 中长期启用它
由于 Guix 具备 reproducible builds, 机会式地使用局域网中邻近的 substitute server 是完全安全的

此外, guix system init 也得到了优化, 这进一步加快了 Guix System 的安装过程

对一些资源受限的机器来说, 构建 Guix 模块本身就是一笔昂贵开销
因此 (guix ci) 模块中新加入了 channel-with-substitutes-available, 它可以把 Guix 拉到 build farm 已经构建完成的最新提交上
具体示例可见 手册

> POWER9, 包, 服务, 以及更多

POWER9 现在以 technology preview 的形式获得支持, 这离不开许多人为 移植 Guix 到该平台 所做的不懈努力
由于我们的 build farm 上 POWER9 容量仍然有限, 当前可用的 binary substitutes 还不算多, 但如果你愿意从源码构建大量软件, 我们非常期待听到你的实际反馈

本次发行新增 2000 个软件包, 总数已超过 17000, 更新了 3100 个软件包
发行版自带 GNU libc 2.31, GCC 10.3, Xfce 4.16.0, Linux-libre 5.11.15, LibreOffice 6.4.7.2 与 Emacs 27.2 等组件
在众多打包变化中, 一个格外醒目的亮点是 新的 OCaml bootstrap, 现在 OCaml 可以通过 camlboot 完全从源码构建
软件更新中还包括 Cuirass 1.0, 也就是驱动我们 build farm 的那套服务

服务目录同样继续扩展, 新增了 wireguard, syncthing, ipfs, 以及 更简单也更方便的 Cuirass 服务 等等
你可以通过 guix system search 来检索这些服务

更多值得关注的变化与 bug fix, 请查看 NEWS

> 试试看

安装脚本 已改进为更适合自动化使用
例如如果你赶时间, 现在可以直接运行:

# yes | ./install.sh

这样便会在没有任何提示的情况下, 直接把 Guix 二进制版本安装到你的系统中

你也可以试试 Guix System 演示虚拟机镜像, 它现在支持与宿主机共享剪贴板, 还支持通过 SPICE 协议动态调整窗口大小, 希望这能进一步改善初次体验

要查看所有可用的安装方式, 请访问 下载页面, 也欢迎通过 联系页面 与我们交流

Enjoy!
