> 开始

运行 info guix 查看在线文档, 或访问 https://guix.gnu.org/manual/stable
查看 https://emacs-guix.gitlab.io/website 获取 Guix 的 Emacs 集成工具

> 指定包

大多数命令都需要一个 包规约(package specification), 下文以 spec 指代, 这里有几个例子:

emacsEmacs 包, 最新版本
gcc-toolchain@7GCC toolchain, 7.x 版本
gcc-toolchain:debug最新 GCC toolchain, 带 debugging symbols

> 管理包

guix search regexp …搜索包
guix show spec显示包的信息
guix install spec…安装包
guix upgrade [regexp]更新包
guix remove name…删除包
guix package -m file从 manifest 实例化包
guix package –export-manifest将当前 profile 导出为 manifest
guix package –roll-back回滚
guix package -I列出已安装的包
guix package -l列出 profile 的 generations
guix package –search-paths显示 search paths
guix package -p profile …使用另外的 profile

> 清单(Manifests)

guix package -m 与其他命令都可以接受一个 manifest 文件, 里面列出你感兴趣的软件包, 形式大致如下:

(specifications->manifest
  '("gcc-toolchain@7" "gcc-toolchain@7:debug" "openmpi"))

> 临时环境

guix shell spec…创建包含 spec... 的环境
guix shell -D spec创建用于开发 spec 的环境
guix shell spec… -C – command …在容器中运行 command …
guix shell –check检查 shell 是否覆盖了环境变量
guix shell -m file为 manifest 文件中的包创建环境

> 更新 Guix

guix describe描述当前的 Guix
guix describe -f channels生成 channel spec
guix pull更新 Guix
guix pull -l查看历史
guix pull –commit=commit更新到指定 commit
guix pull –branch=branch更新到指定 branch
guix pull -C file使用 file 中指定的 channels 更新

> Channel 配置

Channels 指定了 guix pull 应该去哪些 Git 仓库里查找 Guix 及外部包仓库的更新
默认情况下, guix pull 读取 ~/.config/guix/channels.scm, 传递 -C 后则可以读取用户给出的 channel 配置, 例如:

(cons (channel
        (name 'guix-hpc)
        (url "https://gitlab.inria.fr/guix-hpc/guix-hpc.git")
        (branch "master"))
      %default-channels)

> 使用不同版本的 Guix

guix time-machine 命令可以访问 Guix 的其他 revision
比如可以用它安装旧版本的软件包, 或在完全相同的环境中复现一次计算

guix time-machine -C file – commands …
在 file 中给定的 channels 所指定的 Guix 版本下运行 commands …

> 自定义包

guix command name –with-source=name=source用不同的 source URL 构建 name
guix command spec –with-input=spec1=spec2在 spec 的依赖图中用 spec2 替换 spec1
guix command spec –with-graft=spec1=spec2在 spec 中 graft spec2 以替代 spec1
guix command –with-git-url=spec=URL从给定 Git URL 构建 spec
guix command spec –with-branch=package=branch从 package 的指定 Git branch 构建 spec
guix command spec –with-commit=package=commit从 package 的指定 Git commit 构建 spec
guix command spec –with-patch=package=file对 package 应用给定 patch 文件 file 后构建 spec
guix command spec –with-latest=package使用 package 最新的 upstream release 构建 spec
guix command spec –with-c-toolchain=package=toolchain使用 toolchain 构建 package, 再构建 spec
guix command spec –without-tests=package构建 spec 时跳过 package 的测试
guix command spec –with-debug-info=package构建 spec 时为 package 保留 debugging info
guix command spec –with-configure-flag=package=flag构建 spec 时, 为 package 的 configure phase 增加 flag
guix command spec –tune[=arch]按 arch 或当前宿主机架构优化 tunable packages

> 开发包

guix edit spec查看定义
guix build spec …构建包
guix build –log-file spec查看构建日志
guix build -K spec …构建包, 并在失败时保留构建树
guix build -S spec获取 spec 的源代码
guix build –check spec重新构建 spec
guix build –target=triplet spec …为指定 triplet 交叉编译, 例如: arm-linux-gnueabihf
guix download URL从 URL 下载, 并输出其 SHA256 哈希
guix hash file输出 file 的哈希
guix graph spec | dot -Tpdf …查看依赖
guix refresh spec更新包定义
guix import repo name从 repo 导入 name
