我们很高兴宣布 GNU Guix 1.2.0 的发布, 也借此恰好庆祝 Guix 的第八个生日!

本次发行包含:

  • ISO-9660 安装镜像
  • 虚拟机镜像
  • 可在现有 GNU/Linux 发行版上安装包管理器的 tar 包, 既可以 从源码安装, 也可以 从二进制安装

Guix 用户可以通过 guix pull 完成更新

距离上个版本已过去将近 7 个月, 期间有 200 位贡献者提交了代码与软件包, 还有许多人参与了 code review, 系统维护, 翻译, 网站更新, Outreachy mentoring 等重要工作

在这段时间里, 仓库新增了 10200 多次提交, 下面我们会尽量概括其中最值得关注的变化

继续往下读前, 不妨先听听这首特别为本次发行准备的 Ode to One Two Oh, 歌词见 这里

> 安全性

本次发行最重要的亮点之一是支持 channel 认证, 这很可能让 Guix 成为今天交付完整操作系统最安全的方式之一
它补上了我们 “software supply chain” 中缺失的一环, 现在 guix pull 及相关命令会对获取到的 channel 代码进行密码学认证, 例如你将无法从官方 Guix 仓库拉取未授权的提交
关于其设计与实现, 可阅读我们 7 月写的 介绍文章, 手册也分别说明了作为 用户 与 channel 作者 需要了解的内容
另外还新增了 guix git authenticate 命令, 可将同样的认证机制用于任意 Git 仓库

与之配套的是, guix pull 和 guix system reconfigure 现在会检测潜在的 system downgrade 或 Guix downgrade 并直接报错, 从而避免用户被诱导降级到重新引入已知漏洞的软件版本

有了这些保护措施后, 我们还加入了 无人值守升级服务, 简而言之, 它会定期执行 guix pull && guix system reconfigure, 让自动升级与安心感同时到来

另一个我们很自豪的安全改进是将 x86_64 与 i686 上的 binary seed 进一步缩减到 60 MiB, 这得益于 相关工作 以及 GNU Mes, Gash 等软件的持续推进

在同一主题下, build daemon 与 origin 编程接口如今接受新的密码哈希函数, 特别是 SHA-3 与 BLAKE2s, 用于 “fixed-output derivations”, 不再只局限于 SHA256

> 用户体验

让 Guix 对更广泛的用户更易用, 仍然是本次发行的核心目标之一
图形化安装器 与 在其他发行版上安装 Guix 的脚本 都获得了一系列 bug fix 与可用性改进
第一次使用 Guix 的用户会注意到 guix help 现在能更清晰地概览所有命令, guix 默认不再输出过长的下载清单, guix pull 更新 Git checkout 时也会显示 progress bar
另外, guix search, guix system search 以及类似命令现在会自动调用 pager (less 为默认值), 解决了一个频繁被提及的问题

性能方面也有多处提升
Guile 3.0.4 中引入的新 “baseline compiler” 降低了 Guix 自身的构建时间, 这也让 guix pull 的资源占用明显下降
同时在 多个 不同 场景 中也有优化落地, 后续还会继续改进

命令行层面, 新增了 3 个 package transformation options: --with-debug-info, --with-c-toolchain, --without-tests
这些 transformation 如今会记录在 profile 中, 并在 guix upgrade 时自动重放
更重要的是, 它们现在可作用于整个依赖图, 包括 “implicit” inputs, 这让过去做不到的转换也成为可能, 例如:

guix install --with-input=python=python2 python-itsdangerous

新增的 (guix transformations) 模块还提供了 与命令行 transformation 对应的编程接口, 这样你就能在 manifest 中直接使用这些能力

reference manual 也获得了扩展, 新增了 “Getting Started” 章节, “Programming Interface” 一节也为 packager 补充了更多信息
我们在很多地方加入了代码示例, 在线版手册中的标识符还可以直接点击, 跳转到 Guix 或 Guile 手册中的对应位置

最后, 手册现已完整翻译为 法语, 德语 与 西班牙语, 同时还有 俄语 与 中文 的部分翻译
Guix 自身也完整翻译为这 3 种语言, 并在另外 11 种语言上提供了部分翻译

> Packs, GNU/Hurd, 磁盘镜像, 服务

如果你想把 Guix 中的应用带到没有 Guix 的机器上, 那么 guix pack -RR 现在支持新的 “fakechroot” 执行引擎, 并允许通过 GUIX_EXECUTION_ENGINE 在运行时切换不同引擎
对于不支持 unprivileged user namespaces 的宿主机, fakechroot 相较 proot 也能带来 更好的性能

整个系统的交叉编译支持也进一步增强, 例如 guix system build --target=arm-linux-gnueabihf config.scm 这样的命令
再加上软件包与 Guix System 机制层面的移植工作, 现在已经有了 hurd-vm 服务, 可运行 在 GNU/Linux 上以虚拟机形式启动的, 交叉编译版 Guix GNU/Hurd
这也让我们得以开始推进原生 GNU/Hurd 支持

与此相关, 新的 (gnu image) 模块为操作系统镜像提供了更灵活的接口, 在命令行上可通过 guix system disk-image --image-type=TYPE 访问
当前已支持多种 image type, 包括压缩 ISO-9660, 含 ext4 分区的 qcow2, 带 Hurd 选项的 ext2 等, 底层由 genimage 实现

除前文提到的内容外, 系统还新增了十余项服务, 包括 Ganeti, LXQt, R Shiny, Gemini 与 Guix Build Coordinator

本次发行新增 2000 个软件包, 总数已超过 15000, 升级了 3652 个软件包
发行版自带 GNU libc 2.31, GCC 10.2, GNOME 3.34, Xfce 4.14.2, Linux-libre 5.9.3 与 LibreOffice 6.4.6.2 等组件
另外还新增了面向 Maven 软件包的 构建系统, 而在 Guix Days 上, 我们也专门用一场 演讲 介绍了在 Guix 中 bootstrap Maven 的工作

更多值得关注的改动与 bug fix, 请查看 NEWS
